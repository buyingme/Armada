# 12. Glossary

## Domain Terms (Star Wars: Armada)

| Term | Definition |
|------|-----------|
| **Activation** | The process of a ship or squadron taking its turn during the appropriate phase. |
| **Armament** | The dice pool a ship or squadron uses to attack. Includes battery armament (vs ships) and anti-squadron armament (vs squadrons). |
| **Attack Pool** | The set of dice rolled during an attack, determined by armament and range. |
| **Battery Armament** | The colored dice on a ship's hull zone used when attacking another ship. |
| **Command Dial** | A secretly-set dial that determines the command a ship will execute when activated. |
| **Command Token** | A token that can be spent for a weaker version of a command, or stored from a command dial. |
| **Defense Token** | Tokens (Evade, Redirect, Brace, Scatter, Contain, Salvo) that modify incoming attacks. |
| **Engineering Points** | Points used during the Repair command to recover shields or hull. |
| **Fleet** | A player's collection of ships, squadrons, upgrades, and objectives totaling ≤400 points. |
| **Hull Zone** | One of four ship areas (Front, Left, Right, Rear) with independent shields and armaments. |
| **Initiative** | Determines which player activates first; held by the player with fewer fleet points (second player). |
| **Maneuver Tool** | The jointed ruler used to plot ship movement according to speed and yaw values. |
| **Objective Card** | Cards that modify game setup, special rules, and victory conditions. |
| **Round** | One complete cycle of all four game phases (Command, Ship, Squadron, Status). |
| **Speed** | A ship's current movement rate, determining how many joints of the maneuver tool are used. |
| **Squadron Value** | The number of squadrons a ship can activate with a Squadron command. |
| **Yaw** | The number of clicks a joint of the maneuver tool can be adjusted during movement. |

## Technical Terms

| Term | Definition |
|------|-----------|
| **Autoload** | A Godot singleton node that persists across scene changes. |
| **EventBus** | Central signal hub for decoupled inter-system communication. |
| **GDScript** | Godot's built-in scripting language with Python-like syntax. |
| **GUT** | Godot Unit Testing — the testing framework used in this project. |
| **RefCounted** | A Godot base class with reference counting; used for non-scene logic objects. |
| **Resource** | A Godot data container that can be saved/loaded and shared between scenes. |
| **Scene Tree** | Godot's hierarchical structure of nodes that make up the running game. |
| **Signal** | Godot's implementation of the observer pattern for event notification. |
